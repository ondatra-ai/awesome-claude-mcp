# PR Update

Create or update the current branch's PR. Title and body are generated by `claude -p` invoked from inside the script — the agent running this skill does **not** compose them.

## Steps

### 1. Run the bundled script

The script reads the branch's commits and diff vs `origin/main` (falls back to `origin/master`), pipes that context to `claude -p` to generate a conventional-commit PR title and body, then creates a new PR or edits the existing one. It prints the PR URL on completion.

```bash
./.claude/skills/pr-update/pr-update.sh
```

### 2. Post-PR follow-up (when an issue was referenced)

If project docs contain a `**Issue Reference**: [#<n>](...)` link relevant to this branch, also:

1. Check the issue body for an existing `**Related PR**` link.
2. If missing, `gh issue edit <n> --body-file …` to add `**Related PR**: #<pr_number>: <pr_title>`.
3. If present, refresh the existing PR link with the current title.
4. Apply appropriate labels (e.g., `in-progress`, `has-pr`) via `gh issue edit --add-label`.

## Rules

- The script uses `./tmp/` internally and cleans up on success.
- The script's `gh pr view` probe decides create-vs-edit — do not pre-check from SKILL.md.
- The script outputs the PR URL — report that back to the user.
